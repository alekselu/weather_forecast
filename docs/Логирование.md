# Логирование

## Общие принципы

В проекте используется стандартный модуль `logging` Python с централизованной конфигурацией.  
Использование `print()` крайне не рекомендуется.

Логирование должно:

- использовать единый формат
- учитывать уровень логирования
- быть привязано к модулю через `__name__`
- выводиться в stdout (для Docker)

---

## Как добавить логирование в файл

### 1. Импортировать модуль

```python
import logging
````

---

### 2. Создать логгер

```python
logger = logging.getLogger(__name__)
```

---

### 3. Использовать логирование

```python
logger.debug("Отладочная информация")
logger.info("Обычное сообщение")
logger.warning("Предупреждение")
logger.error("Ошибка")
```

---

## Пример использования

```python
import logging

logger = logging.getLogger(__name__)

def process_data():
    logger.info("Начало обработки данных")
    
    try:
        # логика обработки
        logger.debug("Данные успешно обработаны")
    except Exception as e:
        logger.error(f"Ошибка при обработке данных: {e}")
```

---

## Уровни логирования

| Уровень | Назначение                      |
| ------- | ------------------------------- |
| DEBUG   | Детальная отладочная информация |
| INFO    | Основные события работы         |
| WARNING | Потенциальные проблемы          |
| ERROR   | Ошибки выполнения               |

---

## Пример лога

```text
INFO | 2026-05-05 16:49:45,414 | __main__ | successful init
```

---

## Важно

- Не использовать `print()`
- Не вызывать `logging.basicConfig()` вне центральной конфигурации
- Всегда использовать `logging.getLogger(__name__)`
